Instalación paso a paso

1. Clonar el repositorio

git clone https://github.com/tu-usuario/escape-room-3d.git
cd escape-room-3d

2. Crear el entorno virtual

Desde la raíz del proyecto, crea un entorno virtual con Python 3.12:

py -3.12 -m venv entorno311
💡 El nombre entorno311 es el que espera el launcher del juego. Puedes usar otro nombre, pero deberás ajustar la ruta en Frontend/app.py.

3. Activar el entorno virtual

# Windows (CMD)
entorno311\Scripts\activate.bat

# Windows (PowerShell)
entorno311\Scripts\Activate.ps1

# Linux / macOS
source entorno311/bin/activate

4. Instalar dependencias del Backend

pip install -r Backend/req.txt

5. Instalar dependencias del juego

pip install -r game/requirements.txt

6. Instalar dependencias del Frontend

pip install -r Frontend/requirements.txt

7. Aplicar migraciones de base de datos

cd Backend/EscapeRoomPy
py manage.py migrate

8. Crear superusuario (administrador)

py manage.py createsuperuser

Sigue las instrucciones en pantalla para definir nombre de usuario, email y contraseña del administrador.

💡 El superusuario tiene acceso al panel de administración en http://localhost:8000/admin/ y al endpoint /admin-stats/.

Cómo ejecutar

Los tres componentes deben ejecutarse en terminales separadas con el entorno virtual activado.

Backend (Django)

# Desde la carpeta Backend/EscapeRoomPy/
py manage.py runserver

El servidor quedará disponible en http://localhost:8000. Déjalo corriendo mientras usas el dashboard o el juego.

Frontend (Streamlit)

# Desde la raíz del proyecto
streamlit run Frontend/app.py

Se abrirá automáticamente en el navegador en http://localhost:8501.

Juego (pygame)

El juego no se lanza directamente. Se inicia desde el botón "▶ Iniciar Juego" en el tab Jugar del dashboard. Streamlit pasa automáticamente el token JWT y el nombre de usuario al proceso del juego.

⚠️ El backend Django debe estar corriendo antes de iniciar el dashboard o el juego. Sin él, la autenticación y la carga de preguntas fallarán.

Guía de uso del Dashboard

Registro e inicio de sesión

Al abrir el dashboard verás dos tabs: Iniciar Sesión y Registrarse.

  • Para registrarte: introduce un nombre de usuario, email y contraseña, luego haz clic en Crear cuenta.
  • Para iniciar sesión: introduce tu usuario y contraseña, luego haz clic en Entrar.

Una vez autenticado, el token JWT se almacena en st.session_state y se renueva automáticamente cuando expira.

Tabs del Dashboard

🎮 Jugar
Lanza el juego en modo clásico (elige dificultad) o selecciona una Room del catálogo para jugarla.
🗺️ Catálogo
Explora todas las Rooms publicadas. Filtra por dificultad. Cada card muestra título, descripción, creador y número de preguntas.
⚒️ Creator Studio
Crea y gestiona tus propias Rooms. Añade preguntas, publica al catálogo o elimina rooms en borrador.
📜 Mis Sesiones
Historial completo de tus partidas: fecha, tiempo, preguntas respondidas, aciertos y si completaste la sala.
🏆 Tablero
Top 10 de mejores tiempos en el modo clásico. Visible para todos los usuarios.
📊 Estadísticas
Gráficos interactivos de tus partidas: distribución por dificultad, evolución del tiempo, tasa de aciertos.

Guía del juego (pygame)

Al hacer clic en Iniciar Juego, se abre una ventana de 900 × 650 px con la sala 3D. Primero verás la pantalla de título con el nombre de la sala; presiona cualquier tecla para comenzar.

Controles

TeclaAcción
1Seleccionar opción A (primera respuesta)
2Seleccionar opción B (segunda respuesta)
3Seleccionar opción C (tercera respuesta)
4Seleccionar opción D (cuarta respuesta)
ESCSalir del juego en cualquier momento

Objetivo

Responde correctamente el mínimo de preguntas requerido para escapar. El número mínimo se muestra en el HUD inferior (necesitas X/Y). El personaje avanza hacia la puerta verde con cada acierto y retrocede con cada fallo.

HUD (interfaz en pantalla)

  • Barra superior: nombre de la sala, número de pregunta, barra de progreso y cronómetro.
  • Barra de vida (❤): muestra cuántos aciertos llevas respecto al mínimo requerido. Verde = suficiente, amarillo = cerca, rojo = insuficiente.
  • Panel inferior: texto de la pregunta y las 4 opciones de respuesta.
  • Feedback: mensaje verde "✓ ¡Correcto!" o rojo "✗ Era: [respuesta]" tras cada respuesta.

Pantalla de fin

Al responder todas las preguntas, aparece la pantalla de resultado durante 4 segundos antes de cerrarse. Tu tiempo y puntaje se guardan automáticamente en el servidor.

  • ESCAPASTE: respondiste correctamente al menos el mínimo requerido.
  • ATRAPADO: no alcanzaste el mínimo de respuestas correctas.

Solución de problemas comunes

Error de conexión al iniciar sesión

🔌 "No se puede conectar al servidor" — Django no está corriendo. Abre una terminal, activa el entorno virtual y ejecuta py manage.py runserver desde Backend/EscapeRoomPy/.

Token expirado / sesión cerrada automáticamente

⏰ "Tu sesión expiró" — El access token dura 8 horas. El dashboard intenta renovarlo automáticamente con el refresh token (7 días). Si el mensaje persiste, cierra sesión y vuelve a entrar.

La ventana del juego se cierra inmediatamente

🎮 Revisa el archivo game/game_error.log para ver el traceback completo. Las causas más comunes son:
  • Python incorrecto (no 3.12) — verifica con py --version
  • pygame o PyOpenGL no instalados en el entorno virtual activo
  • El backend no está corriendo (no puede cargar las preguntas)

Error de pygame con Python 3.14+

🐍 pygame no es compatible con Python 3.14. Usa exactamente Python 3.12. Crea el entorno con py -3.12 -m venv entorno311.

Las migraciones fallan

🗄️ Asegúrate de estar dentro de Backend/EscapeRoomPy/ al ejecutar py manage.py migrate. Si hay conflictos, prueba con py manage.py migrate --run-syncdb.

Streamlit no encuentra el módulo

📦 Verifica que el entorno virtual esté activado antes de ejecutar streamlit run Frontend/app.py. El comando streamlit debe estar en el PATH del entorno virtual.
