Resumen

La API REST de Escape Room 3D corre en http://localhost:8000. Todos los endpoints devuelven JSON. La autenticación usa JWT Bearer tokens.

LeyendaSignificado
PúblicoNo requiere autenticación
JWTRequiere header Authorization: Bearer <token>
AdminSolo superusuarios Django

Base URL: http://localhost:8000/

Autenticación

MétodoRutaAuthDescripciónRespuesta
POST /token/ Público Obtener par de tokens JWT. Body: {username, password} 200: {access, refresh} / 401: credenciales inválidas
POST /token/refresh/ Público Renovar access token. Body: {refresh} 200: {access} / 401: refresh inválido o expirado

Ejemplo de login

POST /token/
Content-Type: application/json

{
  "username": "mi_usuario",
  "password": "mi_contrasena"
}

// Respuesta 200
{
  "access":  "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

Usuarios

MétodoRutaAuthDescripciónRespuesta
GET /users/ Público Listar todos los usuarios 200: array de usuarios
POST /users/ Público Crear nuevo usuario. Body: {username, email, password} 201: usuario creado / 400: datos inválidos
GET /users/{id}/ Público Detalle de un usuario por ID 200: usuario / 404: no encontrado
PUT /users/{id}/ Público Actualizar usuario completo 200: usuario actualizado / 400: datos inválidos
DELETE /users/{id}/ Público Eliminar usuario 204: eliminado / 404: no encontrado

Sesiones clásicas (GameSession)

Sesiones del modo clásico con preguntas predefinidas.

MétodoRutaAuthDescripciónRespuesta
GET /sessions/ JWT Listar todas las sesiones 200: array de sesiones
POST /sessions/ JWT Registrar nueva sesión. Body: {user, difficulty, time_taken, questions_answered, correct_answers, completed} 201: sesión creada / 400: datos inválidos
GET /leaderboard/ Público Top 10 mejores tiempos (solo sesiones completadas, ordenadas por tiempo) 200: array de {username, time_taken, difficulty, correct_answers}
GET /admin-stats/ Admin Estadísticas globales: total de partidas, tiempo promedio, conteo por dificultad 200: {total_games, avg_time, easy, medium, hard} / 403: no admin

Rooms

Rooms son las salas de escape personalizadas creadas por los usuarios.

MétodoRutaAuthDescripciónRespuesta
GET /rooms/ Público Catálogo de rooms publicadas. Filtro opcional: ?difficulty=easy|medium|hard. Usuarios autenticados también ven sus propias rooms en borrador. 200: array de rooms
POST /rooms/ JWT Crear nueva room. Body: {title, description, difficulty, min_correct}. El creador se asigna automáticamente. 201: room creada / 400: datos inválidos
GET /rooms/{id}/ Público Detalle de una room. Las preguntas no incluyen correct_index para usuarios no autenticados. 200: room / 404: no encontrada
PUT /rooms/{id}/ JWT Actualizar room completa. Solo el propietario puede modificarla. 200: room actualizada / 403: no propietario / 404: no encontrada
PATCH /rooms/{id}/ JWT Actualizar campos parciales de la room. 200: room actualizada / 403: no propietario
DELETE /rooms/{id}/ JWT Eliminar room. No se puede eliminar una room publicada que ya tiene sesiones de juego. 204: eliminada / 403: no propietario / 409: tiene sesiones

Acciones especiales de Room

MétodoRutaAuthDescripciónRespuesta
POST /rooms/{id}/publish/ JWT Publicar room en el catálogo. Requiere entre 4 y 62 preguntas. Solo el propietario puede publicar. 200: room publicada / 400: preguntas insuficientes / 403: no propietario
GET /rooms/{id}/questions/ JWT Obtener todas las preguntas de la room con correct_index. Usado por el game client para cargar la partida. 200: array de preguntas completas / 401: no autenticado
GET /rooms/{id}/stats/ JWT Estadísticas de la room: total de partidas, tiempo promedio, tasa de completado. Solo el propietario puede ver las estadísticas. 200: {total_games, avg_time, completion_rate} / 403: no propietario

Questions

Preguntas asociadas a una Room. Solo el creador puede ver y modificar sus propias preguntas.

MétodoRutaAuthDescripciónRespuesta
GET /questions/ JWT Listar preguntas propias (filtradas por el usuario autenticado) 200: array de preguntas
POST /questions/ JWT Crear nueva pregunta. Body: {room, text, option_0, option_1, option_2, option_3, correct_index} 201: pregunta creada / 400: datos inválidos
GET /questions/{id}/ JWT Detalle de una pregunta propia 200: pregunta / 403: no propietario / 404: no encontrada
PUT /questions/{id}/ JWT Actualizar pregunta completa 200: pregunta actualizada / 403: no propietario
DELETE /questions/{id}/ JWT Eliminar pregunta 204: eliminada / 403: no propietario

Validaciones de Question

  • text: no puede estar vacío, máximo 500 caracteres
  • correct_index: debe ser 0, 1, 2 o 3
  • option_0 … option_3: máximo 200 caracteres cada uno

Room Sessions

Historial de partidas jugadas en rooms personalizadas.

MétodoRutaAuthDescripciónRespuesta
GET /room-sessions/ JWT Historial de sesiones del usuario autenticado, ordenadas por fecha descendente 200: array de sesiones con room_title y user
POST /room-sessions/ JWT Registrar sesión de room. Body: {room, time_taken, questions_answered, correct_answers, completed}. El usuario se asigna automáticamente. 201: sesión creada / 400: datos inválidos

Códigos de error

CódigoNombreCuándo ocurre
200OKPetición exitosa (GET, PUT, PATCH, acciones personalizadas)
201CreatedRecurso creado exitosamente (POST)
204No ContentRecurso eliminado exitosamente (DELETE)
400Bad RequestDatos de entrada inválidos (validación fallida, campos requeridos faltantes, preguntas insuficientes para publicar)
401UnauthorizedToken JWT ausente, inválido o expirado. Credenciales incorrectas en /token/
403ForbiddenEl usuario autenticado no tiene permiso para la operación (no es propietario, no es admin)
404Not FoundEl recurso solicitado no existe
409ConflictConflicto de estado: intentar eliminar una room publicada que ya tiene sesiones de juego

Formato de error

// Error de validación (400)
{
  "title": ["El título no puede superar los 100 caracteres."],
  "difficulty": ["La dificultad debe ser 'easy', 'medium' o 'hard'."]
}

// Error de permiso (403) o conflicto (409)
{
  "detail": "No se puede eliminar una Room publicada que ya tiene sesiones de juego."
}
